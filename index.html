📚 Dynamic Programming Explained

What is Dynamic Programming?

Dynamic Programming (DP) is an optimization technique that solves complex problems by breaking them into overlapping subproblems, storing results to avoid redundant computation.

Overlapping Subproblems

When counting dice sums, the same sub-counts appear repeatedly. For example, ways to reach sum 5 with 2 dice reuse results from 1-dice problems — classic overlap.

Optimal Substructure

The optimal count for n dice and sum s builds from optimal counts for fewer dice: each face value i contributes DP[n-1][s-i].

State Transition

DP[d][s] = Σ DP[d−1][s−i] for i = 1 to 6

Base case: DP[1][s] = 1 if 1 ≤ s ≤ 6, else 0.

Memoization vs Tabulation

Memoization (top-down): Recursion + cache stored results.
Tabulation (bottom-up): Fill a 2D table iteratively — used in this analyzer.

Why DP Beats Recursion

Naive recursion revisits the same (dice, sum) states exponentially. DP computes each state once in O(n × target) time — a dramatic reduction from exponential blow-up.
